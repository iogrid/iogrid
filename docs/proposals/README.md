# Proposals — in-flight design

> **WHAT:** Open design proposals being workshopped. Promote to an immutable ADR in [`../adr/`](../adr/) once accepted.
> **AUTHORITY:** Pre-canon. Reading order: proposal → ADR (if accepted) → architecture/strategy doc absorbs the implication.

## How to add a proposal

Filename convention: `<short-slug>.md`. Open with the problem statement, options considered, recommendation, open questions.

## Lifecycle

```
proposals/<slug>.md  →  adr/NNNN-<slug>.md   (Accepted)
                    \→  archive/<date>-<slug>.md (Rejected — keep history)
```
