# installer/linux/repo — GPG-signed apt/yum/apk repositories

Phase 2 of [EPIC #348](https://github.com/iogrid/iogrid/issues/348). Companion
to the macOS Sparkle work in `installer/macos/sparkle/` (Phase 1, PR #387).

Brings native-package-manager self-update to Linux providers:

```bash
# Debian / Ubuntu
sudo apt update && sudo apt upgrade iogridd

# RHEL / Fedora / Rocky / Alma
sudo dnf upgrade iogridd

# Alpine
sudo apk update && sudo apk upgrade iogridd
```

…and the daemon rolls forward through unattended-upgrades / dnf-automatic.

## Repo URL convention

The Phase 2 publish target is **GitHub Releases assets** as a fallback while
[#392](https://github.com/iogrid/iogrid/issues/392) builds out
`releases.iogrid.org`. Once that endpoint deploys, CI is wired to switch over
by flipping a single env var (`IOGRID_LINUX_REPO_PUBLISH_TARGET`).

| Distro family | Provisional URL (Phase 2)                                              | Target URL (after #392)              |
|---------------|------------------------------------------------------------------------|--------------------------------------|
| Debian/Ubuntu | `https://github.com/iogrid/iogrid/releases/download/linux-repo/deb/…`  | `https://releases.iogrid.org/deb/`   |
| RHEL/Fedora   | `https://github.com/iogrid/iogrid/releases/download/linux-repo/rpm/…`  | `https://releases.iogrid.org/rpm/`   |
| Alpine        | `https://github.com/iogrid/iogrid/releases/download/linux-repo/apk/…`  | `https://releases.iogrid.org/apk/`   |

The path conventions (`/deb/`, `/rpm/`, `/apk/`) match the names in
issue [#390](https://github.com/iogrid/iogrid/issues/390) so the provider-facing
docs in `web/src/app/install/page.tsx` don't have to change when we cut over.

## Trust anchor — single GPG key

A single 4096-bit RSA GPG keypair signs:

- **apt** — detached `Release.gpg` + inline `InRelease` for every `dists/stable/Release`.
- **yum/dnf** — detached `repodata/repomd.xml.asc` (per-arch) + per-package
  signature via `rpm --addsign`.
- **apk** — *not* signed by GPG. Alpine `apk-tools` uses raw RSA-PSS, so the
  apk target uses a separate RSA key generated by `abuild-keygen`. (See the
  Makefile's `apk:` target comment for the rationale + follow-up.)

The keypair is generated by `generate-keypair.sh`:

```bash
./generate-keypair.sh /tmp/iogrid-linux-repo-keys/
# → pubkey.gpg.asc  (commit to repo at installer/linux/repo/pubkey.gpg.asc)
# → privkey.gpg.asc (DO NOT commit; upload to OpenBao + GH Actions secret)
# → keyid.txt
```

### Bootstrapping GitHub Actions

After generating a key:

```bash
# Encode the private key for the CI secret store.
base64 -w0 /tmp/iogrid-linux-repo-keys/privkey.gpg.asc \
  | gh secret set IOGRID_LINUX_REPO_GPG_PRIVKEY_B64 \
      --repo iogrid/iogrid

# Optionally pin the key id (otherwise CI reads it from the imported key).
gh secret set IOGRID_LINUX_REPO_GPG_KEYID \
    --repo iogrid/iogrid \
    --body "$(cat /tmp/iogrid-linux-repo-keys/keyid.txt)"
```

### Bootstrapping OpenBao (backup of the private key)

```bash
bao kv put kv/iogrid/linux-repo/privkey.gpg.asc \
    @/tmp/iogrid-linux-repo-keys/privkey.gpg.asc
bao kv put kv/iogrid/linux-repo/keyid \
    value="$(cat /tmp/iogrid-linux-repo-keys/keyid.txt)"
```

## Client-side provider one-liners

### Debian / Ubuntu

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://releases.iogrid.org/deb/iogrid.gpg.asc \
  | sudo gpg --dearmor -o /etc/apt/keyrings/iogrid.gpg
sudo chmod a+r /etc/apt/keyrings/iogrid.gpg

ARCH="$(dpkg --print-architecture)"
echo "deb [arch=$ARCH signed-by=/etc/apt/keyrings/iogrid.gpg] https://releases.iogrid.org/deb stable main" \
  | sudo tee /etc/apt/sources.list.d/iogrid.list >/dev/null

sudo apt update
sudo apt install -y iogridd
```

### RHEL / Fedora / Rocky / Alma

```bash
sudo rpm --import https://releases.iogrid.org/rpm/iogrid.gpg.asc

cat <<'EOF' | sudo tee /etc/yum.repos.d/iogrid.repo
[iogrid]
name=iogrid — Linux package repo
baseurl=https://releases.iogrid.org/rpm/$basearch/
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://releases.iogrid.org/rpm/iogrid.gpg.asc
EOF

sudo dnf install -y iogridd
```

### Alpine

```bash
sudo wget -O /etc/apk/keys/iogrid@iogrid.org-pubkey.rsa.pub \
    https://releases.iogrid.org/apk/iogrid@iogrid.org-pubkey.rsa.pub
echo "https://releases.iogrid.org/apk/$(apk --print-arch)/" \
  | sudo tee -a /etc/apk/repositories

sudo apk update
sudo apk add iogridd
```

## Key rotation (yearly cadence)

The signing key has a 5-year expiry; we rotate annually as a discipline.

1. Generate a new keypair on a hardened workstation:
   ```bash
   ./generate-keypair.sh /tmp/iogrid-linux-repo-keys-$(date +%Y)/
   ```
2. **Sign the new public key with the old private key** so existing clients
   accept the new trust anchor without manual import:
   ```bash
   gpg --homedir /old/keyring --armor --detach-sign \
       /tmp/iogrid-linux-repo-keys-$(date +%Y)/pubkey.gpg.asc
   ```
3. Replace `installer/linux/repo/pubkey.gpg.asc` in a PR; include the
   signed-by-previous-key signature next to it as `pubkey.gpg.asc.sig`.
4. Rotate the GitHub Actions secret + the OpenBao entry; the next tag push
   will rebuild + re-sign repo metadata against the new key.
5. Leave the **OLD** key in the keyring for 90 days to allow apt clients to
   download both `Release.gpg` (old) and `InRelease` (new) during transition;
   then revoke + purge.

## Local dev workflow

```bash
# Assume you already ran `installer/linux/build.sh` and have artifacts in
# installer/dist/{iogrid_*.deb,iogrid-*.rpm,iogrid_*.apk}.

# 1. Generate a throwaway keypair (don't use this in prod!).
./generate-keypair.sh /tmp/keys-dev/

# 2. Replace the in-repo pubkey for the local build only.
cp /tmp/keys-dev/pubkey.gpg.asc installer/linux/repo/pubkey.gpg.asc

# 3. Build the repo tree.
cd installer/linux/repo
make all \
    IOGRID_VERSION=0.1.0 \
    IOGRID_LINUX_REPO_GPG_PRIVKEY_FILE=/tmp/keys-dev/privkey.gpg.asc \
    IOGRID_LINUX_REPO_GPG_KEYID="$(cat /tmp/keys-dev/keyid.txt)" \
    PKG_DIR=$PWD/../../dist \
    OUT_DIR=/tmp/iogrid-repo-dev/

# 4. Serve over loopback for a smoke test.
( cd /tmp/iogrid-repo-dev && python3 -m http.server 8080 ) &

# 5. From a Docker container, run the apt one-liner with the local URL.
docker run --rm -it --network host ubuntu:24.04 bash -c '
  apt update && apt install -y curl gnupg
  install -m 0755 -d /etc/apt/keyrings
  curl -fsSL http://127.0.0.1:8080/deb/iogrid.gpg.asc \
    | gpg --dearmor -o /etc/apt/keyrings/iogrid.gpg
  echo "deb [signed-by=/etc/apt/keyrings/iogrid.gpg] http://127.0.0.1:8080/deb stable main" \
    > /etc/apt/sources.list.d/iogrid.list
  apt update && apt-cache policy iogridd
'
```

## Phase 2 placeholder warning

The `pubkey.gpg.asc` checked into this directory is a **placeholder**
generated for Phase 2 development; the corresponding private key is NOT
provisioned in the GitHub Actions secret store yet. CI will detect the
missing secret and skip the `publish-repo` job (artifact still uploaded
to the workflow run for manual smoke-testing).
