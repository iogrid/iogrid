iogrid schedules containers to providers with consumer or workstation GPUs.
Two runtimes:

- **NVIDIA on Linux** — CUDA + NVIDIA Container Toolkit
- **Apple Silicon (Mac)** — MLX runtime in an Apple Virtualization Framework
  container, with MLX bindings exposed to Python / Swift workloads

AWS spot for similar VRAM.

preinstalled. iogrid maintains a base image at
`ghcr.io/iogrid/mlx-base:latest`.

## VRAM matching

`min_vram_gb` is the **hard** matching constraint. The scheduler refuses
providers whose GPU memory is below this threshold. Apple Silicon's unified
memory counts toward VRAM (the M2 Max with 64 GB has 64 GB usable VRAM —
unlike a $5K NVIDIA workstation card that ships with the same).

## Gaming-mode preemption

iogrid's Apple Silicon and gamer-PC providers run a **gaming detector**: if
the host's GPU utilization spikes from a non-iogrid process (a video game,
Final Cut, etc.), the iogrid GPU workload is paused within ~3 seconds and the
scheduler attempts to migrate the job to another provider.

For long-running training jobs, this means iogrid is **not the right choice
unless you can checkpoint**. For stateless inference (the dominant use case),
preemption is transparent: the next request lands on a different provider, no
loss of state.

<Aside type="caution" title="Pick the right tool for the job">
  iogrid GPU is great for **stateless LLM/diffusion inference**, batch
  embedding generation, and ML evaluation runs. For multi-day training jobs,
  use a hyperscaler with dedicated capacity. We will not attempt to pretend
  iogrid is the right surface for that.
</Aside>

## Streaming responses

For long-running LLM inference, customers typically need streaming output.
Set `http_endpoint: true` in the workload spec:

```json
{
  "type": "gpu",
  "image": "...",
  "command": ["./serve.sh", "--port", "8000"],
  "http_endpoint": {
    "port": 8000,
    "path": "/v1/chat/completions"
  },
  "gpu_class": "consumer-rtx4090",
  "min_vram_gb": 16
}
```

The response returns a workload-specific endpoint URL:

```json
{
  "id": "wkl_01HXY...",
  "http_endpoint_url": "https://endpoint-01HXY.gpu.iogrid.org/v1/chat/completions",
  "endpoint_token": "tok_..."
}
```

You call that URL like any inference API; iogrid handles the routing,
auth, and per-token billing.
