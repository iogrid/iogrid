iogrid exposes one public API surface with two transports:

- **REST + JSON** at `https://api.iogrid.org/v1/...` — most customers use this
- **gRPC + Connect-Go** at `https://api.iogrid.org` (HTTP/2) — for streaming
  and low-latency clients

Both transports speak the same protobuf schema, defined in
[`proto/iogrid/`](https://github.com/iogrid/iogrid/tree/main/proto/iogrid) and
auto-published as an OpenAPI 3.1 spec.

## Base URLs

| Environment | REST | gRPC |
|---|---|---|
| Production | `https://api.iogrid.org/v1/` | `https://api.iogrid.org` |
| Sandbox | `https://sandbox.api.iogrid.org/v1/` | `https://sandbox.api.iogrid.org` |

Sandbox is fully functional but does not invoke real providers — workload
submissions return canned responses with deterministic timing.

## Versioning

- **v1** — current stable, additive-only changes
- Breaking changes ship under `v2` with a 12-month deprecation overlap
- Protobuf changes go through `buf breaking` in CI; we don't ship breaking
  schema changes within a major version

## Authentication

API keys are bearer tokens passed in the `Authorization` header:

```bash
curl https://api.iogrid.org/v1/me \
  -H "Authorization: Bearer sk_live_abc123..."
```

Keys are prefixed:

- `sk_live_...` — production, bills against your account
- `sk_sandbox_...` — sandbox, no billing
- `pk_live_...` — publishable, safe for browsers, read-only

See [authentication](/api/authentication/) for full details.

## Rate limits

Default limits per key:

| Tier | Requests/min | Burst | Concurrent workloads |
|---|---|---|---|
| Free | 60 | 120 | 5 |
| Pro | 600 | 1200 | 50 |
| Business | 6000 | 12000 | 500 |
| Enterprise | custom | custom | custom |

Limits enforce as a token-bucket — `X-RateLimit-*` headers on every response.
See [rate limits](/api/rate-limits/).

## Errors

All errors return JSON with a stable `code` and a human `message`:

```json
{
  "code": "RATE_LIMITED",
  "message": "Per-minute limit exceeded. Try again in 12 seconds.",
  "request_id": "req_01HXY...",
  "details": {
    "retry_after_seconds": 12
  }
}
```

Full code catalog at [errors](/api/errors/).

## Idempotency

Workload submissions accept an `Idempotency-Key` header (UUID). Retried
requests with the same key within 24 h return the original response, never a
duplicate workload.

## OpenAPI spec

The full machine-readable spec is published on every push to `main`:

- Latest stable: [`/openapi.yaml`](/openapi.yaml) _(generated by CI)_
- Versioned: `https://docs.iogrid.org/openapi-{version}.yaml`
- The reference pages under [`/api/reference/`](/api/reference/) on
  this site are rendered from that file.

<Aside type="tip" title="Postman / Insomnia / Bruno">
  Import the OpenAPI URL directly into your HTTP client; collections regenerate
  automatically.
</Aside>
