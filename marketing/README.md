# iogrid.org — marketing site

Public-facing Next.js 15 app served at https://iogrid.org. Static export, deployable to any static host (Cloudflare Pages, Vercel, S3+CloudFront, the OpenOva k8s nginx, etc.).

This app is **separate** from the management plane at `web/`. The management plane handles signed-in flows (providers, customers, billing). This app handles unauthenticated marketing, blog, legal, and install funnels.

## Stack

- Next.js 15 App Router, `output: "export"` for fully static HTML
- TypeScript 5.7 strict
- Tailwind 3.4 (CSS variables for theming, brand tokens consumed from `../brand/tokens/`)
- Inter + JetBrains Mono via `next/font/google`
- shadcn/ui patterns reused but no client-side primitives shipped yet (everything is server components for the static export)
- Sharp for OG image generation

## Local development

```bash
pnpm install
pnpm dev                      # → http://localhost:3000
```

## Build + static export

```bash
pnpm build                    # writes ./out/
```

The `out/` directory contains all static HTML, CSS, JS, and assets. Deploy by copying it to any static host.

## OG image

```bash
pnpm og:gen                   # regenerate public/og-image.png
```

Re-run whenever the brand or hero tagline changes.

## Brand tokens

The Tailwind config (`tailwind.config.ts`) imports `../brand/tokens/colors.json` and `../brand/tokens/spacing.json`. To change a color or spacing token, edit those JSON files — they're the single source of truth and the `web/` management plane will consume the same values.

## Lighthouse target

We target ≥95 on all categories on every page. CI runs Lighthouse against a built export on each PR (see `.github/workflows/marketing-ci.yml`).

## What this app does not do

- No client-side data fetching (deliberately — keeps everything static and fast)
- No auth, no forms POSTing to backends (CTAs deep-link to `app.iogrid.org`)
- No analytics tag, no Google Tag Manager (we'll add self-hosted PostHog in Phase 1)
- No newsletter signup (deferred — we'll add a self-hosted Listmonk in Phase 1)

## Linking conventions

- `/install/` renders the all-platforms downloads table with SHA-256s.
- `/install/<os>[/<arch-or-format>]` are statically-generated meta-refresh redirect pages to the signed installer artefacts on `releases.iogrid.org`. Slug table lives in `app/install/[...slug]/page.tsx`.
- The download manifest is single-sourced in `content/installer-manifest.ts`; the build script writes it out to `public/install/manifest.json` plus one `.sha256` file per artefact for the documented `curl … .sha256 | shasum -a 256 -c -` verification flow.
- Auth and dashboard CTAs link to `https://app.iogrid.org/...` (the management plane).
- Docs CTAs link to `https://docs.iogrid.org/...` (separate docs site, Phase 1).
